Project Qualee

This is the second case of the three case studies I was assigned during my internship at FABA.

Task overview

I worked with the Qualee project manager to draft up the content. Then I proceeded to brainstorm and experiment different practices to achieve the storytelling style. I was also in charge of creating the mockups posted along with the case study.

Let’s take a look

Project overview

Cloud-based onboarding and engagement platform with micro-learning modules and customized experiences.

Problem

Recognizing that Qualee were experiencing difficulties in finding the right dedicated offshore team, FABA brought an attentive and committed team to work with the company. Our team constantly improves, and builds features for the website and app.

Solution

We worked with Qualee to improve the existing codebase and build new features for an application that allows companies to engage their new hires in a faster and more efficient way by: Inspecting the existing code and refactor the code to improve the product’s performance. Rebuilding the architecture for more future feature extensions. Re-organizing the working process.

The Process

First, we agreed on a short-term test on the performance of our development team. With time of the essence, our team jumped right in to patching and adding core features. Meanwhile, we analyzed the code in order to draft a technical plan to reduce technical debt and improve maintainability.

After we delivered satisfactory results, Qualee successfully raised funds for the project and decided to build a team of eight dedicated members based on Faba’s standards. We share an accompanying relationship in development.

An offshore team can result in a breakdown of communication both ways but FABA has experience removing such concerns from the equation. By arranging daily standups between the development team and the Qualee team, the project has been kept in smooth operation since the getgo. Additionally, the project manager also has weekly progress check-ins with the client to ensure the project is adhering to the roadmap, and addressing any issues before they arise.

Features

  • A demo version is provided when downloading Qualee App.
  • A structured journey of the company on the home page.
  • Multimedia library with content supported not only in text but also in audio, documents, images & videos.
  • Multi-language: supported in 16 of the world’s most widely used languages.
  • Messages: contact with others for support, with secure chat where users can begin anonymous messaging.
  • Offline mode: allows users to access the library content without an internet connection.

Technologies

  • Laravel for API, Admin.
  • Swift for iOS.
  • Java for Android.
  • AWS’s server.
  • MySQL for database.
  • Stripe gateway for payment.
  • HTML5/CSS3/JQuery/Javascript for front-end website."
